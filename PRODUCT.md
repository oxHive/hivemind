# Product

## Register

product

## Users

Developers who use Claude Code and other MCP-compatible AI agents. They run HiveMind locally as a background service and interact with the dashboard to inspect, manage, and review what their agents remember. They're comfortable in the terminal, operate in dark environments, and have high standards for information density and clarity.

## Product Purpose

HiveMind is a local-first persistent memory store for AI coding agents. The dashboard is the control surface: browse and edit stored memories, review the relationship graph, triage feedback and conflicts, and manage server and sync settings. Success looks like a developer who never has to think about the UI — it's a quiet, reliable tool that stays out of the way.

## Brand Personality

Calm, structured, capable. The interface should feel like a well-run filing system: everything in its place, nothing shouting for attention. Confidence comes from clarity, not from decoration.

## Anti-references

- Generic SaaS dashboards: metric cards, gradient accents, chart-heavy layouts, the Vercel/Datadog aesthetic. HiveMind is infrastructure, not analytics.
- VS Code extension panels: too boxy, too much indirection, feels like an afterthought bolted onto a bigger app. The dashboard is a first-class product.

## Design Principles

1. **Disappear into the task.** The tool should be invisible. No decorative motion, no chrome that draws the eye away from content.
2. **Density as respect.** Developers can handle information. Prefer compact layouts that show more over spacious layouts that show less.
3. **State over decoration.** Every visual signal earns its place by communicating state (selected, loading, error, syncing). No color, border, or animation for aesthetics alone.
4. **Earn familiarity.** Follow established product UI conventions so users spend zero time learning the interface. Surprise belongs in the product, not the UI.
5. **Dark is the default.** This is a local dev tool used in dark environments. The dark theme is not a preference — it's the identity.

## Accessibility & Inclusion

Best effort. No hard WCAG target, but readable contrast and keyboard accessibility are expected. Text must be legible, interactive elements must be focusable, and the interface must not rely on color alone to convey meaning.
